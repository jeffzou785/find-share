### § 7.2 缺失数据不应简单抬高分数

**判定：✅ 保留 — 当前确实有此问题**

实际审查 `src/screening/scoring.py:260-268`：

```python
total_w = 0.0
weighted_sum = 0.0
for score_val, weight_val in subscores.values():
    if score_val is None or weight_val <= 0:
        continue
    weighted_sum += score_val * weight_val
    total_w += weight_val
base = weighted_sum / total_w if total_w > 0 else None
```

**问题**：缺失项的权重被排除，剩余项权重重新归一化。举例：某海外公司没有研报数据（catalyst=None）+ 没有被忽视证据（neglect=None），原权重 `growth 0.25 + val 0.20 + qual 0.20 + cat 0.15 + neg 0.20 = 1.0`，缺失 catalyst+neglect 后变成 `0.65/0.65`，**剩余三个子分权重被人为抬高 54%**。

improvements 担心正确——**信息少的公司反而分数相对偏高**。

修复建议：
1. 不重新归一化，缺失项按 0.5 中性填充；
2. 或输出 `final_score` + `confidence = (已得分项权重和 / 1.0)`，让消费者自己决定是否信任。

> 反质询：improvements §7.2 的修复方向（输出 score + confidence 双字段）是对的，但**当前 `ScoreMetrics` 已经有 `weights_used`，可以反推 confidence**——不必新加字段，只需在 `ScoreMetrics` 加 `coverage_ratio: Optional[float]`（已得分项权重和）。工程量 0.5 天。

## 七、策略二（医药量价齐升）对抗式审查

> 审查立场：A 股医药生物（申万一级，约 470 家上市公司）的特殊性远高于消费和制造业出海——它同时受 (1) 医保集采的"以价换量"结构性压制、(2) 创新药 FDA/EMA 出海的"事件驱动"催化、(3) DRG/DIP 全院控费、(4) 商保+消费医疗新主线、(5) License-out 一次性收入等多条独立逻辑影响。improvements §3 用单一"量价齐升"模型涵盖这么多矛盾，是这版文档最大的设计风险。

### 7.0 整体定性：策略二在错误的抽象层

**improvements §3 的根本问题是把"医药"当成一个行业来筛——这在 A 股不成立。**

- 化药（仿制药）的核心矛盾是**集采**（已 10 批，仿制药收入承压）。
- 创新药的核心矛盾是**临床读出 + License-out + FDA 批准**（2023-2025 年是出海大年，恒瑞、百济、科伦博泰、和黄、信达、再鼎等）。
- 医疗器械的核心矛盾是**DRG/DIP 控费 + 局部集采**（骨科、心血管、IVD）。
- 医疗服务 / 消费医疗（眼科、齿科、医美、辅助生殖）的核心矛盾是**消费周期 + 商保政策**，与医保集采几乎无关。
- CXO 的核心矛盾是**海外投融资周期 + SAFE 名单**，improvements §3.2 自己排除了。
- 中成药的核心矛盾是**OTC 渠道 + 中药集采**（湖北牵头批次），与化药集采节奏完全不同。

improvements §3.2 把"医美消费医疗"和"纯创新药"排除，**等于主动放弃 2025-2026 年 A 股医药最大的两个 alpha 来源**：
- **创新药出海**：恒瑞（DLL3/HER2 ADC 多笔 License-out）、科伦博泰（MSD 多笔）、百济（百悦泽 FDA）、和黄（呋喹替尼 FDA）、传奇（Carvykti FDA）、信达（信迪利 FDA）等。
- **消费医疗复苏**：2024 年底消费医疗基数已低，2025 年企稳反弹（爱尔、通策、华熙、爱美客等）。

> 反质询：作者说"纯创新药不应和仿制药/器械集采修复混在一起"——**这句话的结论是对的（不能混），但处理方式是错的**。正确处理是把策略二拆成 2-3 个独立子策略，而不是排除创新药。排除创新药 = 放弃最大 alpha。

### 7.1 §3.1 第一性原理的硬伤：集采是"以价换量"，"量价齐升"在 A 股医药极其罕见

**improvements §3.1 自己引用的三个事实——价格下降、销量提升、集中度提升——已经决定了"量价齐升"是少数派。** 在集采语境下：

- **价格**：中标价平均降幅 50-80%（化药）、60-70%（骨科）、50-70%（IVD），**结构性下降**。
- **量**：中标企业市占率提升，但**总市场（按金额）通常萎缩**。
- **毛利率**：价格下降倒逼成本优化，但毛利率仍多数下滑。

只有以下三种 case 才能真正"量价齐升"：
1. **创新产品**（无集采压力，定价自由）。
2. **出海产品**（海外价格不受国内集采影响）。
3. **集采后产品结构升级**（老产品降价 + 新产品放量，结构性改善，少数公司能做到）。

**improvements §3.5.2 把"价稳/价升"作为通用筛选条件，会把第 1、2 种以外的所有集采修复股过滤掉**，而第 3 种（结构升级）极其罕见。

> 反质询：作者能不能在 A 股医药里举出**近 3 年 10 个真实"量价齐升"案例**？我列举几个公认案例（恒瑞 2024、华东医药 2023、迈瑞医疗器械出海、心脉医疗、惠泰医疗），共同点都是**"创新+出海"驱动**，**不是"集采修复"驱动**。improvements 的"集采修复→量价齐升"逻辑链，在 A 股历史上**支撑不起 10 只以上的样本集**。

### 7.2 §3.3"集采出清"是事后判断，不是事前 signal

improvements §3.3 把"集采影响出清"作为筛选条件，但：

- **集采出清是事后判断**：集采第 N 批落地后，企业收入承压 2-3 年，渠道去库存 1-2 年，新品放量 1-2 年——**总修复周期通常 3-5 年**。财报"出清"信号出现的时点，往往已经是修复周期中后期，股价已反映。
- **集采批次差异巨大**：化药 1-3 批（2018-2020，降价最猛）、4-7 批（2021-2023，节奏稳定）、8-10 批（2023-2024，规则变化）；骨科关节（2021，第一轮）、脊柱（2022，第二轮）、运动医学（2023，第三轮）；IVD 生化（2022）、化学发光（2023，安徽为主）。**每一批的修复节奏都不同，不能用统一时间窗口**。
- **improvements §3.3"过去 4-8 个季度经历过下行"= 1-2 年**：太短，**漏掉真正的修复周期股**（化药第 1-3 批的修复到 2023-2024 才出现，需要 4-5 年）。建议改为"过去 8-16 个季度"（2-4 年），且按集采批次配对（公司最近一次集采批次的落地年份）。

> 反质询：作者能否说清楚"华东医药 2023 年业绩修复"对应的是**哪一批集采**（米氮平、吲达帕胺等在第 3 批 2020 年）？批次的"出清时点"如何用代码识别？这条不做对，整个 §3.3 三层判断就是空话。

### 7.3 §3.7 数据源映射全错——医药的真实 ground truth 在 CDE/FDA/联采办，不在 iwencai

improvements §3.7 列了 9 个数据源，**没有一个是医药行业的真实 ground truth**：

| improvements §3.7 列的数据源 | 实际问题 |
|---|---|
| 巨潮公告 + 年报 PDF | 财报 lag 严重（年报 4 月、半年报 8-9 月），出清信号出现时往往滞后 2-3 季度 |
| 新浪三表 | 只有汇总财务，无产品级收入/ASP/出货量 |
| mootdx 财务快照 | 同上 |
| 腾讯财经 PE/PB | 通用，无医药特殊字段 |
| 东财研报 | 研报覆盖偏向大票（恒瑞、迈瑞），中小盘器械/创新药覆盖极少 |
| **iwencai 研报语义搜索** | **不是医药行业的核心数据源**——iwencai 返回的是网页搜索结果，不是结构化数据 |
| 东财个股新闻 | 通用，缺乏药品批件/注册/招标维度 |
| 同花顺热点 | 通用 |
| 股东户数/融资融券/资金流 | 通用筹码信号 |

**真实 ground truth 数据源（improvements 完全没提）：**

1. **国家组织药品联采办（上海阳光医药采购网）**：集采中标结果、采购量、降幅——这是**判断"集采受损→修复"的唯一权威数据源**。可爬取。
2. **NMPA / CDE 药品审评**：药品注册申请、临床试验默许、附条件批准、突破性治疗——判断创新药进展的权威数据。`cde.org.cn` 可爬。
3. **FDA Drugs@FDA / Orange Book**：ANDA/NDA/BLA 批准、专利到期——创新药出海的权威信号。免费 API。
4. **ClinicalTrials.gov**：临床读出时间、入组人数、主要终点——创新药事件驱动信号。
5. **医药魔方 / Citeline / GBI数据库**：License-out 交易对手、首付、里程碑——但**收费**，A 股散户拿不到。可降级用公告抓"授权协议"。
6. **各省招标采购中心**：骨科、IVD、心脏支架等区域集采（省级 + 联盟）——比国家联采更细。
7. **DRG/DIP 分组方案**：医保支付方式改革对器械的影响（较粗，但 2024 年起部分省有数据）。

> 反质询：作者想做"集采出清证据"，**有没有接过上海阳光医药采购网的中标数据？** 没有的话，§3.6 全部关键词都得从财报 PDF + 研报里抓，**等于把数据问题降级成 NLP 问题**。NLP 在医药财报上的准确率不会超过 70%（"集采""出清""放量"等词在每份医药年报里都出现，**模板话术污染严重**）。

### 7.4 §3.6 "集采出清证据"是模板话术，文本信号的边际信息量极低

improvements §3.6 列了 13 个正面关键词（"集采影响出清"、"价格趋稳"、"中标放量"...）。

**实证问题**：A 股医药年报从 2021 年开始，**几乎所有受集采影响的公司年报都会同时出现这些词**。"集采影响逐步消化"、"中标产品放量"、"以价换量"——这些是**年报董事会在必写话术**，不构成 signal。

> 反质询：作者用这些关键词扫描 100 份医药年报，**命中率会接近 100%**——这意味着边际信息量 ≈ 0。真正的 signal 应该是：
> - "集采产品收入同比 +X%"（数字，而非"放量"）
> - "毛利率同比提升 X pp"（数字）
> - "中标省份从 N 扩大到 M"（具体）
> - "重点产品市占率从 X% 提升到 Y%"（具体）
>
> **抓数字而不是抓形容词**。improvements §3.6 的关键词全是形容词，应该全部改为数字模式（正则：`(\d+\.?\d*)\s*%`+ 上下文）。

### 7.5 §3.8 打分模型的设计错误：周期低位 vs 集采出清 高度相关

improvements §3.8 给出权重：
- 周期低位 20%
- 集采出清 20%
- 量 20%
- 价 20%
- 财务质量 10%
- 催化 10%

**两处设计错误：**

**错误 1**：周期低位（PE/PB 分位）和集采出清（基本面事件）**在 A 股医药高度正相关**——集采受损股的 PE 自然位于历史低位。**给同一信号打两次分 = 双重计算**。

**错误 2**：量 + 价共 40%，但 improvements §3.5 自己承认"如果没有销量和价格数据，用收入+毛利率代理"——这意味着 40% 权重的两个子分，**实际上是同一个代理指标**（`revenue_yoy` + `gross_margin_yoy`）。等于 40% 权重压在 2 个财报字段上，**任何一个噪声都会主导打分**。

> 修复建议：
> - 合并"周期低位 + 集采出清"为单一"周期阶段"子分（25%）。
> - 量价齐升不应拆为独立子分，应作为**质量子分**的一个维度（与现金流、毛利率合并，共 30%）。
> - 新增"产品组合"子分（创新药占比、海外收入占比、集采产品占比）——这才是医药股的真正区分度。

### 7.6 §3.4 硬筛条件会漏掉关键修复股，且阈值过严

| §3.4 条件 | 问题 |
|---|---|
| PE 分位<40% | 修复初期 PE 高（业绩刚回正），分位会更高；改成 PE 分位<60% + 趋势改善 |
| PB 分位<60% | 创新药/器械 PB 天生高（轻资产+研发估值），这条会系统性排除创新子行业 |
| 股价距 3 年高点回撤>40% | 修复初期股价已反弹 30-50%，回撤收窄；这条会漏掉已开始反弹的修复股 |
| 收入 yoy>10% | 太高——集采修复初期收入 yoy 通常在 5-15%（如恒瑞 2024 收入 yoy ≈ 13%）。改成 5% |
| 扣非 yoy>20% | **致命**：集采修复股的扣非 yoy 经常 -20% 到 +20% 之间（业绩刚回正），20% 阈值会把所有早期修复股排除。改成 "扣非 yoy > -10% 且连续两期改善" |
| 毛利率同比下降<1pp | OK，但应同时允许"环比改善" |
| OCF/净利>0.6 | OK |
| 资产负债率<60% | OK |

> 反质询：作者能不能用 §3.4 的硬筛在恒瑞医药（600276）2023 年报数据上跑一遍？我赌命中不了——恒瑞 2023 年扣非 yoy ≈ +21%（刚到线），但毛利率下降 1.5pp（不满足），股价距 3 年高点回撤仅 30%（不满足）。**真正的修复龙头会被这套硬筛漏掉**。

### 7.7 §3.2 排除 CXO 的逻辑可疑——A 股 CXO 2025-2026 有独立 alpha

improvements §3.2 写"CXO：主要受投融资周期和海外订单影响，不是典型集采修复逻辑"——**这句话在事实上对，但结论（排除 CXO）可疑**：

- CXO 在 2025-2026 年的 alpha 来自：(1) 美联储降息→生物医药投融资回暖、(2) 国内创新药 License-out 增加→国内 CXO 订单增加、(3) SAFE 名单移除（药明康德）。
- 这是**与"集采修复"独立的另一条 alpha 主线**。
- 正确处理：CXO 单独成策略（出海型 + 投融资周期修复型），不混在策略二里——**但也不应完全放弃**。

> 类似的，**"医药商业"**（上海医药、国药一致、大参林、益丰等）的修复逻辑是院外渠道+DTP 处方外流，**也是独立主线**，improvements 简单排除掉。

### 7.8 §3.9 输出字段——`vbp_exposure` / `volume_signal` 等关键字段定义模糊

improvements §3.9 新增字段：

- `vbp_exposure`（集采暴露度）：**怎么算**？是集采产品收入占比？是过去 3 年集采中标金额？是集采批次加权？作者没说。
- `volume_signal`（量信号）：是布尔值、连续值还是枚举？基于什么数据？
- `price_signal`（价信号）：同上。
- `product_mix_signal`（产品组合信号）：同上。

**这些都是"听起来对但无定义"的字段**。落地时会发现：
- `vbp_exposure` 需要该公司每款产品的集采状态（中标/落标/未纳入），**这个数据公开不可得**（要爬联采办 + 各省招标 + 公告拼凑）。
- `volume_signal` 在大部分医药股（无具体出货量披露）下只能 fallback 到 `revenue_yoy`，**与 `revenue_yoy` 字段重复**。

> 反质询：作者能否给出 `vbp_exposure` 在恒瑞医药 2024 年报上的具体计算公式和数值？给不出来就说明这个字段是占位符，**不应写进 §3.9 的输出 schema**。

### 7.9 集采"受益股"vs"受损股"未区分

improvements §3.1 提到"竞争格局重塑，中标企业集中度提升"，**但 §3.4 硬筛条件里没有"中标状态"字段**。

- 集采对中标企业（如恒瑞、华东医药、信立泰）是**受益**（市占率提升）。
- 对落标企业（如部分原研药企、未中标的国产仿制药企）是**受损**。
- 对未纳入集采的企业（如部分创新药、OTC、消费医疗）**无影响**。

策略二如果只筛"集采修复股"，**应该明确锁定"曾经落标或集采产品收入占比>30%的企业"**，否则会把从未受集采影响的标的（创新药、医美）误纳入"修复"主题。

> 反质询：作者如何区分"集采修复股"和"从未受集采影响的医药股"？两者财务表象可能高度相似（收入增长 + 毛利率稳定），但内在驱动力完全不同。

### 7.10 商誉减值 / 研发资本化是医药股核心雷区，但 §3.6 只列为关键词

A 股医药的财务雷区集中在两点（**可数字量化**，不是 NLP 问题）：

1. **商誉减值**：步长制药、康恩贝、海思科、华润三九等历史并购形成的商誉/净资产比例过高。
2. **研发资本化**：恒瑞（一直费用化，是行业标杆）、复星医药、丽珠等部分资本化（粉饰利润）。资本化率>50% 是危险信号。

improvements §3.6 把这些列为文本关键词（"商誉减值或研发资本化过高"），但**实际数据完全可以从三表算出来**：

```
商誉/净资产 = balance_sheet.goodwill / balance_sheet.equity_attributable_to_parent
研发资本化率 = cashflow.capitalized_rd / (income_statement.rd_expense + cashflow.capitalized_rd)
```

> 建议：把这两条升级为**硬过滤**（商誉/净资产>50% → rejected；研发资本化率>50% → watch），不是 NLP 关键词。

### 7.11 医药行业池未在当前项目代码中定义

当前项目 `src/strategies/` 只有 `consumer_reversal.py` 和 `overseas_champion.py`，**没有医药策略文件**。`TARGET_INDUSTRIES` 列表里**没有任何医药相关行业**。

策略二落地前，**前置工程**：

1. 新增 `src/strategies/pharma_vbp_recovery.py`（策略二骨架）。
2. 在 `industry_mapping.py` 加医药行业映射（申万一级"医药生物"+ 二级"化学制剂/化学原料药/中药/生物制品/医疗器械/医疗耗材/医药商业/医疗服务"）。
3. 决定哪些二级子行业纳入策略二（建议：化学制剂、中成药、医疗耗材、医疗器械、IVD——创新药和 CXO 独立成策略）。

**improvements §3.2 的"适用范围"列了 7 个细分方向（化学制剂/中成药/生物制品/医疗器械/医疗耗材/IVD/骨科等），但没有给出对应的申万二级行业代码**——这是落地第一步，作者跳过了。

### 7.13 创新药出海主线应该独立成策略（建议拆出策略二B）

**这是本次审查最强烈建议的修改**：把策略二拆成两个子策略。

#### 策略二A：集采修复型（仿制药/器械/耗材/IVD）

- 适用：化学制剂、中成药、医疗耗材、IVD、骨科、心血管（部分）。
- 核心：曾经落标或集采产品占比高 → 收入/毛利率开始修复。
- 数据源：联采办中标数据 + 财报三表 + 公司公告。
- 关键字段：`vbp_product_count`（公司集采中标产品数）、`vbp_revenue_share`（集采产品收入占比）、`recovery_quarter_count`（修复持续季度数）。
- 量价齐升：弱（多数以价换量），核心看收入 + 份额。

#### 策略二B：创新药出海型（创新药/创新器械）

- 适用：生物制品（创新药）、部分医疗器械（高端影像、机器人、AI 辅助诊断）。
- 核心：FDA/EMA 批准、License-out 首付、海外商业化收入。
- 数据源：FDA Drugs@FDA + CDE + ClinicalTrials.gov + 公告（授权协议）+ 财报（海外收入附注）。
- 关键字段：`license_out_count_12m`（近 12 月 License-out 笔数）、`license_out_upfront_yi`（首付总额，亿元）、`fda_approval_count`（FDA 批准产品数）、`overseas_revenue_yoy`（海外收入同比，与策略三共用字段）。
- 量价齐升：强（创新药 ASP 高，海外定价不受集采影响）。

**两条主线的财务特征、估值方法、催化剂完全不同，不应合并打分**。

> 反质询：作者能否在 improvements §3 现有框架下，同时容纳"恒瑞医药（仿制药修复 + 创新药出海双主线）"和"春立医疗（纯骨科集采修复）"？前者需要 License-out + FDA 字段，后者需要集采中标 + 份额字段——**统一打分模型必然对其中一类失衡**。

### 7.14 策略二的真实样本量估计

A 股医药生物约 470 家上市公司。按 improvements §3.4 硬筛 + §3.2 适用范围：

- 行业池（化学制剂 + 中成药 + 生物制品 + 医疗器械 + 耗材 + IVD）：约 320 家
- 非 ST、上市满 3 年、市值≥30 亿：剩约 220 家
- PE 分位<40% + PB 分位<60%：剩约 50 家
- 收入 yoy>10% + 扣非 yoy>20% + 毛利率下降<1pp：剩约 15-25 家
- OCF/净利>0.6 + 负债率<60%：剩约 10-15 家

**最终命中约 10-15 家**，与策略三当前 15 只相当。但 10-15 个样本上做 6 个子分 + 风险扣分 + 4 层判断（周期低位 / 出清 / 量 / 价），**过拟合风险与策略一（4 只）相当**。

> 建议：先在 P0 阶段把硬筛跑通，拿到实际命中数。如果 <10 家，应该**先扩样本（放宽阈值）**，再做打分模型；不能在 10 家样本上拟合一堆权重。

### 7.15 §9.1 P0 第 6 项"固化集采修复样本"是 P0 唯一可立即开干的策略二任务

improvements §9.1 P0 列了第 6 项"固化集采修复样本：至少覆盖化学制剂、器械、耗材、IVD 等方向"。

**这条对，但定义不清**。"固化样本"意味着：

- 标注每只的"集采批次 + 修复起点季度 + 修复持续季度数 + 股价表现"。

### 11.2 第一性原理追问（"响应版本"自身的新问题）

> 即便 review 意见被完整响应，新 improvements.md 本身仍要接受对抗。以下 10 条是新版引入或未解决的新风险。

#### 追问 1：响应得"太完整"，是不是另一种形式主义？

review 提了 30+ 条意见，improvements 几乎**全盘接受**。但全盘接受 = 没有自己的判断。

**例**：review §7.6 建议"删掉 PB 分位<60%"（理由：创新药 PB 天生高）。improvements §5.2 直接照单全收"不强制 PB 分位"。但**策略二A 是集采修复型**（仿制药/器械/IVD），不是创新药——这些公司 PB 没那么高，PB 分位<60% 的硬筛其实**合理且有用**。

**盲目接受 review 可能误伤策略二A 的精度**。作者应该有"反驳 review"的环节，而不是"采纳 review"。

#### 追问 2：策略二B 在 A 股（不含港股）的样本量，作者算过吗？

improvements §5.3 适用范围：创新药、生物制品、创新器械。

**现实问题**：A 股（不含港股）真正意义上的"创新药"公司**不到 30 家**——恒瑞、信达、百济、复星、君实、康方、再鼎、艾力斯、泽璟、迪哲、益方、海创、荣昌、康诺亚、和黄、贝达、微芯等。

其中真正能在 §5.3 催化剂下产生"License-out + FDA 批准"信号的，**<10 家**（且大部分核心标的在港股或科创板，A 股主板极少）。

**improvements 没说策略二B 在 A 股的样本量预估**——如果潜在标的 <10 家，建独立子策略 + 接入 CDE/FDA 数据（4-6 周工程量）的 ROI 是否合理？**或许应该把策略二B 推迟到港股通接入后再做**。

#### 追问 3：联采办/CDE/FDA 数据接入的工程量被严重低估

improvements §10.2 P1 第 8-9 项把"接入联采办 + CDE/FDA/ClinicalTrials"放 P1。

**真实工程难度**：
- **联采办（上海阳光医药采购网）**：无官方 API，需要爬虫 + 反爬应对；中标结果按批次发布，需要回溯 10 批化药 + 4-5 批器械。
- **CDE 药品审评**：网站结构化查询难，药品审评数据格式不统一（化药/生物药/中药三套）；需要处理"申请人 / 适应症 / 临床分期"等多对多映射。
- **FDA Drugs@FDA / Orange Book**：英文，需要标准化药品名映射（公司中文名 ↔ FDA 应用号 ≠ 一一对应，同一个药在不同公司有不同代号）。
- **ClinicalTrials.gov**：英文 + 医学专业术语，NCT 号与公司公告的对应关系需要人工或 LLM 桥接。

**保守估计每个数据源 1.5-2 周，3 个数据源 = 5-6 周**。improvements §10.2 P1 总共列了 11 项"2-4 周"——**P1 时间预算至少低估 50%**。

#### 追问 4：ground truth 30 只样本的标注一致性没说

improvements §5.4 第 2 项 + §10.1 P0 第 10 项：建 30 只历史样本 CSV，标"集采批次 / 修复起点季度 / 修复持续季度数 / 股价表现 / 是否真修复"。

**问题**：5 个字段里 3 个是**主观判断**：
- "修复起点季度"：是收入同比转正的那季？毛利率止跌的那季？还是扣非转正的那季？**不同人不同答案**。
- "修复持续季度数"：取决于"修复起点"的定义，**误差累积**。
- "是否真修复"：**最主观**——华东医药 2023 算修复吗？恒瑞 2024 算修复吗？春立医疗 2024 算修复吗？

**单人标注 1-2 周，但主观标签的 inter-rater reliability 是问题**。improvements 没提：
- 标注规则文档（rule book）放在哪里？
- 多人交叉标注 + 分歧仲裁机制？
- 标签版本管理（后续发现标错怎么改）？

**没有 rule book 的 ground truth = 主观偏见数据集，训练/校准出来的阈值还是主观的。**

#### 追问 5：覆盖率报表的"覆盖率"定义模糊

improvements §7.3 列了 9 项要统计（"成功获取财务数据数"、"成功获取估值数据数"...）。

**问题**："成功获取"的判定标准是什么？

- 新浪财报三表拉到了，但缺 `eps_basic` 字段——算"成功获取财务数据"吗？
- `source_status.financials = "ok"` 但 `eps_basic = None`——这是"成功"还是"部分缺失"？
- PE 历史拉到了 800 天但样本不足 100 天（不满足 `min_history_samples`）——算"成功"吗？

**improvements 没说覆盖率是行级还是字段级**。这会导致：
- 同一份报表，不同人打出不同覆盖率。
- 跨 run 比较覆盖率时，定义漂移导致趋势失真。

**建议**：明确"成功率 = 满足策略硬筛所需字段全部非空的样本数 / 应处理样本数"，字段级而非行级。

#### 追问 6：logging 替换 print 的工程量被低估

improvements §10.1 P0 第 6 项："引入 logging，替换主要 print"。

**真实工程量**（review 之前估的"1 天"也偏乐观）：
- 20 处 print 分布在 collectors / strategies / scripts 三层。
- 替换为 logging 不只是改 import：
  - 定义 logger 层级（collectors.akshare_impl / strategies.consumer_reversal / screening.scoring 等）
  - 决定每条 print 是 INFO / WARN / ERROR / DEBUG
  - 决定日志格式（timestamp + module + run_id + message）
  - 决定日志文件路径和轮转策略
  - 决定 tqdm 进度条与 logging 共存方式（tumblr/stqs 兼容）
- **实际 2-3 天**，且需要重跑一次完整 pipeline 验证日志完整。

**P0 时间预算因此进一步偏乐观**。

#### 追问 7：P0 的 10 项任务，1-2 周真的能完成？

improvements §10.1 P0 列了 10 项，预算"1-2 周"。逐项累加：

| # | 任务 | 真实工程量 |
|---|---|---|
| 1 | 修 risk_penalty bug | 1 小时 |
| 2 | 修评分归一化 + coverage_ratio | 1 天 |
| 3 | 数据覆盖率报表 | 2 天 |
| 4 | 补 hot_reason_count_30d | 2 天 |
| 5 | 补 relative_return_60d | 2 天 |
| 6 | 引入 logging | 2-3 天 |
| 7 | 扩研报到 200 篇 | 8-10 小时（脚本就绪，主要是网络拉取） |

#### 追问 8："消费子行业差异化阈值暂缓"没有退出条件

improvements §4.3 暂缓子行业差异化，理由"4 只样本过拟合"。

**但消费股子行业差异极大**：食品饮料（高毛利+品牌）vs 家电（出口+地产链）vs 纺服（库存周期）vs 零售（同店）vs 社服（经营杠杆）——**一套阈值必然对某些子行业系统性误杀**。

§10.3 P2 第 7 项说"评估是否需要消费子行业差异化阈值"——**评估条件是什么？reject 分布？标签数据？**

**没有明确退出条件，"暂缓"会变成"永久不做"**。建议：

```
退出条件：当消费股 reject 分布显示某子行业（如家电/纺服）被系统性剔除 > 50%
         且该子行业有 ≥ 5 只样本带 human_label
         且整体消费股命中率 ≥ 15 只（样本充足）
         → 自动启动子行业差异化阈值设计
```

这才是可执行的"暂缓 → 启动"路径。

#### 追问 9：策略二B 与港股的时序错配

A 股创新药标的中，**百济神州、信达、再鼎、康方、和黄**等同时在港股上市。

- 港股披露 FDA 进展、临床读出、License-out 公告**往往早于 A 股公告**（港交所披露易 + 英文公告更及时）。
- 策略二B 如果只看 A 股财报 + 巨潮公告，**会错过港股披露更早的信息**，导致策略二B 的"催化剂"信号滞后 1-2 周。
- improvements §5.3 + §7.1 完全没提**A+H 股的时序错配**。

**建议**：策略二B MVP 阶段明确"只覆盖纯 A 股创新药（如恒瑞、复星、贝达、康缘、华东等）"，A+H 股暂缓到港股通/披露易接入后再做。否则会出现"信号源不一致"的方法论 bug。

#### 追问 10：CXO/医药商业/医美"单独成策略二C/二D"是画饼

improvements §5.5 提到"CXO、医药商业、医美未来新增策略二C/二D，但不应污染集采修复模型"。

**但 §10 执行计划里完全没有提到策略二C/二D**——P0/P1/P2 三层都没有。

**这是"画饼"还是"暂缓"？措辞不清**。如果是暂缓，应该列入 §10.4"暂缓 / 删除"表，给出明确的"何时启动"条件（如"策略二A/二B 跑通 + 命中数稳定后"）。
